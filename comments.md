# 自然中文注释

与 [SKILL.md](SKILL.md) 配套。**本文件是 Java 类注释模板、方法/步骤注释、语气与去 AI/机翻味的唯一权威**。Mapper XML、YAML 细则分别见 [data-access.md](data-access.md)、[config-yml.md](config-yml.md)。

## 适用范围

| 产物 | 规范位置 | 要点 |
|------|----------|------|
| Java | 本文件 | 五层结构、步骤分块、语气 |
| Mapper XML | data-access.md | SQL 方法标签正上方注释 |
| YAML | config-yml.md | 段分割符、key 旁注释 |

共用原则：**写业务口径与边界，不写复述**；**自然中文，无机翻味、无 AI 套话**；改逻辑必改注释。

## 硬性原则

- **新增/修改业务代码、XML、YAML**：默认补注释
- **可省略**：getter、简单 setter、纯重命名、单行 typo、仅格式化且无语义变化
- **改逻辑必改注释**
- **朗读自检**：像同事口头交代 → 合格；翻译腔、说明书、AI 套话 → 重写

## 注释五层结构

| 层级 | 形式 | 写什么 |
|------|------|--------|
| **类** | 见下节「类注释」 | 职责 + 边界（事务、幂等、口径） |
| **方法** | 多行 `/** ... */` | 做什么、触发条件、返回值；`@param` / `@return` |
| **步骤块** | `// 动词+对象+边界` | 本步目的；不满足怎么办；**块间空一行** |
| **块内行** | `//` 单行 | 口径、单位、分支原因、并发/幂等 |
| **字段** | `/** ... */` | 业务含义、单位、外部系统对应 |

步骤分块结构与 Service 模板见 [method-structure.md](method-structure.md)。

**禁止**用阿拉伯数字步骤注释：

```java
// 获取订单
// 判断状态
```

<a id="类注释硬性"></a>

## 类注释（硬性）

项目 `CLAUDE.md` 有专属模板时**服从项目**；无模板时用下列标准三要素：

```java
/**
 * @description: 订单服务实现
 * @author: 开发者
 * @since: 2026-06-09 10:00
 */
```

| 标签 | 要求 |
|------|------|
| `@description` | 一句话类职责；写业务口径或边界，不写 CRUD 功能列表 |
| `@author` | 创建人 |
| `@since` | `YYYY-MM-DD HH:mm` |

**变体**（切面、工具类）：

```java
/**
 * 衍生数据汇总，计算口径与统计 SQL 一致。
 *
 * @author 开发者
 * @since 2026-06-09 09:36
 */
```

`@description` 为空或缺 `@author` / `@since` → 不合格。

## 方法 Javadoc

- 第一句：完整职责
- 第二句（如有）：边界、幂等、跳过条件
- `@param` / `@return`：业务含义；空/0 代表什么

## 步骤分块与块内注释

长方法、公开业务方法**必须**步骤分块。格式与好/差对照见 [method-structure.md](method-structure.md)；块内示例：

```java
// 按分组键归并，数量取组内最小值（与统计口径一致）
// 单条失败不挡整批，方便断点续跑
```

| 好 | 差 |
|----|-----|
| `// 解析条件表达式，不通过则不执行` | `// 解析` |
| `// 无记录或无明细时清空衍生数据` | `// 判断订单` |
| `// 用待支付状态作更新条件，防并发重复` | `// 更新状态` |

<a id="语气与排版"></a>

## 语气与排版

- 短句、直说；DTO、幂等等术语保留英文
- 中英空格：`按 orderId 幂等更新`
- 全角标点用于中文句

### AI 套话 — 见到就删或改

| 删掉或改写 | 替换思路 |
|------------|----------|
| 该方法用于… / 本方法的主要作用是 | 直接写职责 |
| 值得注意的是 / 综上所述 | 删除 |
| 旨在 / 有效地 / 进行…操作 | 删修饰，保留动词 |
| 确保…的正确性 | 写具体规则或删除 |
| `// 首先…其次…最后…` | 改步骤分块，每步独立 `//` |

### YAML / XML 共用语气

`该配置项用于…`、`本节主要用于…`、复述 key/SQL 关键字 → 改为主动短句说明业务用途。YAML 正反例见 [config-yml.md](config-yml.md)。

## 润色示例

**机翻 Javadoc → 自然中文：**

```java
// 前：值得注意的是，本方法主要用于处理支付回调。
// 后：支付回调；同一 outTradeNo 重复通知时幂等返回，不重复更新状态。
```

**复述代码的行注释 → 业务口径：**

```java
// 前：// 遍历列表
// 后：// 按分组键归并后写回汇总表
```

更多见 [examples.md](examples.md)、[pitfalls.md](pitfalls.md#五注释反面教材)。

## 与代码同步

改逻辑必改注释。Java / XML / YAML 口径变更时同步更新对应注释。
