# 自然中文注释

与 [SKILL.md](SKILL.md) 配套。**注释是硬性约束**：新增或修改业务代码默认必须补注释。项目 `CLAUDE.md` / rules 中的类注释模板优先于本文件。

## 步骤分块标杆

方法体分块与步骤注释，对齐 [method-structure.md](method-structure.md) 中的**通用模板**：

1. 方法 Javadoc 概括全流程
2. 体内按步骤分块，每步一行总注释
3. 块间空一行
4. 总注释写「做什么 + 边界/跳过条件」
5. 块内复杂逻辑再补单行注释

## 硬性原则

- **新增/修改业务代码**：默认必须补注释
- **可省略**：getter、简单 setter、一眼能看懂的赋值
- **改逻辑必改注释**；注释与实现不一致视为缺陷
- **朗读自检**：像同事口头交代规则 → 合格；像翻译腔 → 重写

## 注释五层结构

| 层级 | 形式 | 写什么 |
|------|------|--------|
| **类** | `@description` + `@author` + `@since` | 职责 + 关键边界（事务、幂等、口径） |
| **方法** | 标准多行 `/** ... */` | 整段做什么、触发条件、返回值；`@param` / `@return` |
| **步骤块** | `// 动词+对象+边界` | 本步目的；不满足时怎么办；**块间空一行** |
| **块内行** | `//` 单行 | 口径、单位、分支原因、并发/幂等 |
| **字段** | `/** ... */` | 业务含义、单位、与外部系统对应 |

**禁止**：

```java
// 1. 获取订单
// 2. 判断状态
// 查询数据库
// 遍历列表
```

## 方法 Javadoc（必须详细）

至少包含：

- 第一句：完整职责（不只重复方法名）
- 第二句（如有）：边界、前置状态、幂等/跳过条件
- `@param`：业务含义
- `@return`：返回什么、空/0 代表什么

```java
/**
 * 业务成功后解析上下文，按配置执行同步或清理。
 *
 * @param joinPoint 切点
 * @param marker    业务标记注解
 */
public void afterSuccess(JoinPoint joinPoint, BusinessMarker marker) { ... }

/**
 * 重算并落库衍生数据。
 * 主记录不存在或明细为空时清空；有数据则先删后插。
 *
 * @param id 业务主键
 */
public void rebuildById(Long id) { ... }
```

## 步骤分块注释（方法体内，重点）

长方法、公开业务方法**必须**步骤分块：

1. 每步块首一行 `//` 总注释
2. **块与块之间空一行**
3. 格式：**动词 + 对象 + 边界/跳过条件**
4. 不用 `----` 装饰线
5. 块内复杂时补单行注释

```java
public void afterSuccess(JoinPoint joinPoint, BusinessMarker marker) {
    // 解析条件表达式，不通过则不执行
    if (!evaluateCondition(marker.condition(), joinPoint)) {
        return;
    }

    // 解析业务主键列表
    List<Long> ids = resolveIds(marker, joinPoint);
    if (ids.isEmpty()) {
        return;
    }

    // 按动作类型执行同步或删除
    for (Long id : ids) {
        ...
    }
}
```

### 步骤注释：好 vs 差

| 好 | 差 |
|----|-----|
| `// 解析条件表达式，不通过则不执行` | `// 解析 condition` |
| `// 无记录或无明细时清空衍生数据` | `// 判断订单` |
| `// 先删后插，保证与源数据一致` | `// 保存数据` |
| `// 用待支付状态作更新条件，防并发重复` | `// 更新状态` |

## 块内行注释

```java
// 按分组键归并，数量取组内最小值（与统计口径一致）
// 期数至少为 1，避免除零
// 单条失败不挡整批，方便断点续跑
```

## 类注释（硬性）

```java
/**
 * @description: 业务同步切面；方法正常返回后触发，抛异常则不同步
 * @author: 开发者
 * @since: 2026-06-09 14:19
 */
```

复杂类可在 `@description` 或首行补充边界，例如「切面 Order 设在事务拦截器内侧」。

`@description` 为空视为不合格。

## 字段注释

```java
/** 金额，单位：分 */
/** 外部流水号，与第三方支付 out_trade_no 一致 */
```

## 语气与排版

- 短句、直说；术语不硬翻（DTO、SpEL、幂等保留英文）
- 中英空格：`按 orderId 幂等更新`
- 全角标点用于中文句子
- 去 AI 味、去机翻味（见下表）

## 应避免的模式

| 模式 | 改法 |
|------|------|
| 只有 Javadoc、体内无步骤分块 | 补步骤注释与空行 |
| 步骤注释只有名词 | 补动词与边界 |
| 机翻被动语态 | 改主动短句 |
| 复述可见代码 | 写业务口径或删除 |
| AI 套话 | 删除 |
| 用 `----` 装饰线 | 删除，靠空行分层 |

## AI 套话 — 见到就删或改

| 删掉或改写 | 替换思路 |
|------------|----------|
| 该方法用于… / 本方法的主要作用是 | 直接写职责或规则 |
| 值得注意的是 / 综上所述 | 删除 |
| 旨在 / 有效地 / 进行…操作 | 删修饰，保留动词 |
| 确保…的正确性 | 写具体规则或删除 |

## 润色示例

### 体内无分块 → 步骤分块

**前：**

```java
public void rebuildById(Long id) {
    if (Objects.isNull(id)) return;
    Order order = orderService.getById(id);
    if (Objects.isNull(order)) { deleteById(id); return; }
    ...
}
```

**后：** 见 [method-structure.md](method-structure.md) Service 方法模板。

### 步骤注释太简略

**前：** `// 解析`

**后：** `// 解析业务主键列表`

### 支付回调幂等

**前：**

```java
/**
 * 值得注意的是，本方法主要用于处理支付回调。
 */
```

**后：**

```java
/**
 * 支付回调处理；同一 outTradeNo 重复通知时幂等返回，不重复更新订单状态。
 */
```

## 正反例

### 好

- 步骤分块 + 块间空行 + 边界写进步骤注释
- 块内补充口径：`// 单条失败不挡整批，方便断点续跑`

### 差

- 只有 Javadoc，体内无分块、无空行
- `// 查询未删除的`、`// 遍历列表`（复述代码）
- `// 首先…其次…最后…`（AI 三段式）
