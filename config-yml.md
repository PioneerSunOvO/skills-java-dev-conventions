# YAML 配置注释

与 [SKILL.md](SKILL.md)、[comments.md](comments.md) 配套。适用于 Spring Boot / Spring Cloud 项目的 `application.yml`、`application-{profile}.yml`、`bootstrap.yml` 及项目约定的配置目录。

**配置修改路径以当前仓库邻文件为准**；不散落修改用户未点名的模块配置。

## 硬性原则

- **按段组织**：不同模块、不同顶层路径之间，用**段落分割符**隔开；简述本段配置作用
- **新增或修改配置项**：默认补中文注释，说明业务用途与非自解释取值
- **语气**：自然中文，无机翻味、无 AI 套话——**权威规则见 [comments.md#语气与排版](comments.md#语气与排版)**
- **安全**：禁止在注释或示例中写真实密码、私钥、token

## 段落分割符（硬性）

不同顶层路径（如 `server`、`spring`、`spring.cloud`、`自定义前缀`）之间，**必须**用分割符隔开，便于检索与维护。

### 格式

```yaml
############################## <本段作用简述> ######################################
<本段配置内容>
```

| 要素 | 要求 |
|------|------|
| 简述 | 中文短语，说清本段管什么，如「服务端口与上下文」「数据源」「注册中心」 |
| 长度 | 宜短（约 4～12 字），不写「本节用于配置…」 |
| 成对 | **不要求** `start/end` 成对出现；**有分割符即可** |
| 长度一致 | 分割符整行（含中文简述）长度与邻段保持一致；同文件内尽量统一（例如都用同样数量的 `#`，不足部分用 `#` 补齐） |
| 位置 | 紧贴本段首尾；段间可空一行 |

### 何时必须加分隔

- 新增独立顶层配置块
- profile 文件中新增与 `spring` 平级的大块
- 框架配置与业务自定义配置之间切换

### 段内小标题

段内子模块用一行 `#` 注释即可，不必再额外加“段落分割符”：

```yaml
############################## 第三方支付 ########################################
app:
  pay:
    # 渠道 A
    channel-a:
      notify-url: ...
    # 渠道 B
    channel-b:
      notify-url: ...
```

### 正反例

**好**：

```yaml
############################# 服务端口与上下文 ###################################
server:
  port: 8080
  servlet:
    context-path: /api

############################## 业务自定义配置 #####################################
app:
  feature:
    enabled: true
```

**差**：仅用单行 `# 下面是 server` 划段；分割符无具体简述；分割符长度忽长忽短；段与段黏连无分割符。

## 必须写注释的场景

| 场景 | 写什么 |
|------|--------|
| 新增自定义业务配置块 | 段头分割符 + 模块职责 |
| 回调 URL、外部系统地址 | 用途、环境、对应接口 |
| 金额/数量限额 | 单位、`-1` 等特殊取值 |
| 开关影响业务范围 | 开/关行为差异 |
| Spring Cloud | 服务名、命名空间、配置中心 dataId 等业务含义 |
| 多 profile 差异 | dev/test/prod 行为说明（不写密钥） |

## 可简短或合并说明

- 标准 Spring 配置且无项目定制口径
- 同组字段含义相同（一组注释覆盖）
- 语义明确的布尔开关

## 段内注释形式

- 简单取值：行尾 `# 说明`
- 规则多：key 上方块注释

禁止复述 key 名、禁止注释中写真实密钥。

## 与 Java / XML 分工

- **YAML**：段分割、配置项用途、环境、单位
- **Java**：`@ConfigurationProperties` 绑定后的业务规则、匹配优先级
- **Mapper XML**：SQL 口径

改配置语义时同步更新分割符简述与段内注释。

## 交付自检

- [ ] 不同顶层路径有分割符，简述可读
- [ ] 分割符长度一致（含中文简述），与邻段对齐
- [ ] 新增 key 有必要说明
- [ ] 无密钥明文；语气见 comments.md
